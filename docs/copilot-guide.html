Customization Guide

How to use the agents, skills, prompts, and instructions created for the AI Face Detection project — detect real vs. AI-generated faces with EfficientNet-B0.

How it works

All customizations live under .github/ and extend GitHub Copilot's behavior in VS Code. There are four types:

TypeWhat it doesHow you invoke itLocation
Agent A persona with specific tools and constraints. Routes complex, multi-step tasks. Type @AgentName in chat, or pick from the agent selector .github/agents/
Skill Domain knowledge loaded on-demand. Keeps context lean — loaded only when relevant. Type /skill-name in chat, or invoked automatically by an agent .github/skills/<name>/SKILL.md
Prompt A parameterized template for a focused, single-purpose task. Type /prompt-name in chat .github/prompts/
Instruction Always-on style rules applied automatically to matching files. Automatic — activates when you edit a matching file .github/instructions/
💡

Tip: You don't need to memorize invocation syntax. In the Copilot chat panel, type @ to browse agents and / to browse skills and prompts.

⚠️

Why keep Training separate? It's the only agent with execute access (runs shell commands). Keeping it isolated prevents editing or writing agents from accidentally running code.

Skills

Skills are domain-knowledge bundles loaded on-demand. Invoke them directly with /skill-name, or let @PaperWriter route to them automatically.

Skill report-drafting .github/skills/report-drafting/SKILL.md

Drafts or fills any section of the 15–20 page academic report. Reads real metrics from reports/ and artifacts/ before writing — knows the exact file names to look for (eval_results.json, train_log.csv, etc.). Uses [INSERT: …] placeholders when files don't exist yet. Never fabricates numbers. Saves output to reports/report_draft.md.

Instructions: code-style

Always-on style rules that activate automatically whenever you edit Python files matching src/**/*.py, scripts/**/*.py, or notebooks/**/*.py. No invocation needed.

📏 code-style.instructions.md Instruction

Enforces PyTorch + timm conventions, Dataset/DataLoader patterns, training loop idioms, evaluation practices, file organization, naming conventions, and reproducibility (seed setup).

.github/instructions/code-style.instructions.md

Key rules enforced

AreaRule
DeviceAlways use a device variable; never hardcode cuda:0
Mixed precisionUse torch.cuda.amp.autocast() + GradScaler
timmCreate with num_classes=0, attach custom head separately
LossBCEWithLogitsLoss — do not apply sigmoid before loss
CheckpointsSave state dicts only, not full model objects
MetricsEvaluate on test set; use sklearn.metrics
SeedsSet random, numpy, torch, cuda seeds at script start
NamingPascalCase classes · snake_case functions · UPPER_SNAKE constants
ConfigAll hyperparameters and paths from YAML in configs/, never hardcoded
ℹ️

Automatic: You don't need to do anything. When you open or edit a .py file under src/, scripts/, or notebooks/, Copilot will follow these rules automatically.

Claude Code — CLAUDE.md

If you use Claude Code (Anthropic's agentic CLI / VS Code extension) instead of or alongside GitHub Copilot, the .github/ customizations are not read. Claude Code uses a flat CLAUDE.md file at the project root instead.

🤖 CLAUDE.md Claude Code

A single Markdown file at the project root that Claude Code reads automatically on every session. Contains the complete project context: dataset paths, model architecture, training stages, HP tuning rules, code conventions, safety rules, and output file locations — distilled from all .github/ customizations.

CLAUDE.md

How Claude Code reads CLAUDE.md files

File locationScopeRead automatically?
CLAUDE.md (project root) ✅This projectYes — every session
src/CLAUDE.md, scripts/CLAUDE.md, …That subdirectory onlyYes — when Claude navigates there
~/.claude/CLAUDE.mdAll projects, all machinesYes — user-level global
.claude/CLAUDE.md ❌—No. .claude/ is for settings only
ℹ️

The .claude/ directory holds settings.json (tool permissions, allowed commands) and settings.local.json (gitignored local overrides) — not the CLAUDE.md instruction file.
